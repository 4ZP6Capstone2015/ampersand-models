Instructie voor gebruiken nieuwe frontend + framework met demoapp CB

Merk op! Nog niet alle functionaliteit die in de huidige prototypes zit is verwerkt in deze nieuwe frontend.

Stappen:
1. Check-out 'frontend' branche van Git repo
2. Copy volledige inhoud van map 'frontend' naar 'htdocs\CB\' (CB is hier de naam van applicatie, straks te benaderen via http://localhost/CB)
2. Genereer ampersand script CB.adl (includes CB.pop en Planner.ifc)
	* Genereer naar: 'htdocs\CB\ampersand\'
	* Gebruik: --proto --no-static-files
3. Ga naar http://localhost/CB
4. Kies 'Extensions -> Reset Database' (door middel van middelste icoontje aan de rechterkant)
5. Have fun!

Probeersels die de huidige features demonsteren:
- Ga naar interface 'Projects':
	- Hier vind je een voorbeeld van de TABS implementatie.
	- De projecten staan in een COLS implementatie.
	- Verwijder de naam van een project. Bij het verlaten van de focus van het element (onBlur) komt de invariant overtreding omhoog en wordt een rollback gedaan. Elke wijziging levert direct communicatie met de backend op. Dit gaat de basis worden om interface rules te kunnen evaluren en de exec engine on the fly formulieren aan te vullen. Functionaliteit is nog niet af.
	- Probeer een project te verwijderen uit de lijst, middels het rode kruisje helemaal achteraan de rij. Dit mag niet, want deze expressie "V[SESSION*Project];projectStarted;'true';projectStarted~" in deze interface is not editable. Merk op!
